## Payment‑Gateway Comparison & Recommendation

| Option | Typical Flow | Docs / Setup Needed | Effort (hrs) | Ease of Integration | Fees | Recommended For |
|--------|--------------|---------------------|--------------|---------------------|------|-----------------|
| **Razorpay (Web Checkout)** | Server creates order → client opens Razorpay modal → signature verification → mark asset owned | Razorpay Dashboard (KYC, API keys), server SDK (`npm i razorpay`), webhook URL | 8‑12 | Medium (SDK provided) | ~2 % + ₹3 per txn | Primary choice – unified UI for cards, UPI, Google Pay; already partially stubbed in code |
| **PhonePe (Web SDK)** | Server creates signed payload → client redirects to PhonePe → callback verification | PhonePe Business account, Merchant ID & Secret, docs for JSON payload & HMAC | 10‑14 | Medium‑High (no UI component, manual redirects) | ~1.9 % + ₹2 | If you have a large PhonePe user base and want native experience |
| **Google Pay via Razorpay** | Same Razorpay flow; just enable GPay in dashboard | Razorpay settings only | No extra dev | Low | Same as Razorpay | Add‑on to Razorpay for GPay users |
| **Direct UPI Deep‑Link** | Generate `upi://pay?...` URI → open on mobile → bank app processes → callback URL | Your VPA from any Indian bank, simple URI builder | 4‑6 | Low (very minimal code) | No gateway fees (bank fees only) | Low‑value purchases, fallback option |
| **Paytm (Web Checkout)** | Similar to Razorpay: order creation → Paytm JS SDK → checksum verification | Paytm Business account, MID & Merchant Key, SDK | 8‑10 | Medium | ~1.75 % + ₹3 | If you need strong Paytm user coverage |

### Recommended Approach for ZenMind
1. **Adopt Razorpay** as the primary payment gateway. It already aligns with the existing stub code, supports **cards, UPI, Google Pay**, and provides robust server‑side verification and webhooks.
2. **Enable Google Pay** in the Razorpay dashboard (no extra code) to capture GPay users.
3. **Optional fallback:** Add a simple **Direct UPI deep‑link** button for low‑price assets where you want zero‑fee transactions.
4. **PhonePe** can be introduced later if analytics show a high concentration of PhonePe users.

---
### What You Need to Provide / Prepare
- **Razorpay Business Account** – completed KYC, bank settlement details.
- **API Keys** – `RAZORPAY_KEY_ID` and `RAZORPAY_KEY_SECRET`; store them in your `.env` (backend & front‑end if needed).
- **Webhook URL** – a public HTTPS endpoint (e.g., `https://yourdomain.com/api/webhooks/razorpay`) registered for events like `payment.captured` and `order.paid`.
- **SSL Certificate** – required for webhook and client‑side checkout.
- **Server SDK** – `npm i razorpay` (if not already installed).
- **Frontend Integration** – replace the fake‑payment modal with Razorpay Checkout:
  ```ts
  const options = {
    key: data.keyId,
    amount: data.order.amount,
    currency: data.order.currency,
    name: 'ZenMind Wellness Store',
    description: asset.title,
    order_id: data.order.id,
    handler: async (resp) => {
      await apiFetch(`/store/${asset._id}/verify`, {method: 'POST', body: JSON.stringify(resp)});
      // unlock download, show toast, etc.
    },
    prefill: {email: user.email, contact: user.phone},
  };
  new window.Razorpay(options).open();
  ```
- **Backendpoints**:
  - `POST /store/:id/purchase` – creates Razorpay order and returns `{keyId, order}`.
  - `POST /store/:id/verify` – verifies the HMAC signature (`crypto.createHmac('sha256', KEY_SECRET)`).
- **Testing** – Use Razorpay sandbox credentials, test card numbers (`4242 1111 1111 1111`) and UPI IDs (`test@upi`).
- **Production Switch** – swap sandbox keys for live keys and ensure webhooks are active.

### Step‑by‑Step Checklist (Razorpay Implementation)
1. Create/verify Razorpay account and add bank details.
2. Generate API keys; add to `.env` (`RAZORPAY_KEY_ID`, `RAZORPAY_KEY_SECRET`).
3. Install SDK: `npm i razorpay`.
4. Implement `/store/:id/purchase` (order creation) on backend.
5. Implement `/store/:id/verify` (signature verification) on backend.
6. Register webhook URL in Razorpay dashboard.
7. Update front‑end `WellnessStore.tsx` – replace fake modal with Razorpay Checkout call.
8. Enable Google Pay in Razorpay settings (optional).
9. (Optional) Add Direct‑UPI button that builds `upi://pay?...` URL.
10. Test end‑to‑end in Razorpay test mode.
11. Deploy with live keys and monitor webhooks.
